## 페이지 / 권한 구조

일반 사용자 페이지, 상인 페이지, 관리자 페이지는 3개로 분리한다. 로그인은 가능하면 통합하고, 로그인 후 `USER`, `MERCHANT`, `ADMIN` Role에 따라 접근 가능한 화면과 API를 제한한다.

```
일반 회원가입
→ USER 계정 생성
→ 상인 등록 신청
→ 관리자 승인
→ role이 MERCHANT로 변경
→ 상인 페이지 접근 가능
```

| Role | 화면 범위 |
| --- | --- |
| USER | 일반 사용자 기능, 프로필, 찜, 리뷰, 결제 |
| MERCHANT | USER 기능 포함, 내 가게 관리, 메뉴 관리, 광고 신청, 정산 신청 |
| ADMIN | 관리자 대시보드, 유저 관리, 상인 승인/거절, 가게 관리, 신고 처리, 광고/정산/인증 심사 |

## 16. 공통 UI 컴포넌트

| 컴포넌트 | 설명 | 사용 화면 |
| --- | --- | --- |
| **하단 네비게이션 탭** | 홈/지도/검색/채팅/마이 5탭 | 메인 화면 전체 |
| **토스트 메시지** | 성공/실패 알림 (하단 고정, 3초 후 사라짐) | 전체 |
| **바텀시트** | 상세 정보, 필터, 옵션 메뉴 | 지도, 커뮤니티 등 |
| **확인 팝업** | 삭제/정지 등 취소 불가 작업 전 확인 | 전체 |
| **로그인 유도 팝업** | 비로그인 사용자 인증 필요 기능 접근 시 | 찜, 리뷰, 채팅 등 |
| **로딩 스피너** | API 호출 대기 중 화면 | 전체 |
| **빈 상태 화면** | 데이터 없을 때 안내 메시지 + 액션 버튼 | 목록 화면 전체 |
| **이미지 슬라이더** | 장소/상품 이미지 좌우 스와이프 | 장소 상세, 스토어 |
| **별점 컴포넌트** | 1~5점 별점 입력/표시 | 리뷰, 동행 리뷰 |
| **엽전 배지** | 🪙 아이콘 + 금액 표시 | 스토어, 결제, 마이 |
