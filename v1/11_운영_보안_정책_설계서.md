## 1. CORS 정책 및 보안 설정 정의서

---

### 📌 1-1 출처(Origin) 관리 정책

환경별로 허용 Origin을 분리한다. 운영 환경에서는 보안상 와일드카드(`*`)를 사용하지 않는다.

| 환경 | 허용 Origin | 비고 |
| --- | --- | --- |
| Local | `http://localhost:3000`, `http://localhost:5173` | React/Vite 로컬 개발용 |
| Dev/Staging | 추후 배포 도메인 확정 시 반영 | 개발/스테이징 서버 |
| Prod | 추후 운영 도메인 확정 시 반영 | 실제 서비스 운영 도메인 |
- 운영 환경에서는 `https://`만 허용한다.
- `allowCredentials=true`를 사용하는 경우 `allowedOrigins=*`는 사용할 수 없다.
- Refresh Token은 HttpOnly Cookie로 전달하고, Access Token은 응답 Body로 전달한다.

---

### 📌 1-2 인증 및 식별 (Credentials) 정책

- **`allowCredentials` 정책:** `true`
- **목적:** 프론트엔드와 서버 간 쿠키, Authorization 헤더를 통한 인증 정보 교환 허용.
- **제약 조건:** 이 값이 `true`일 경우 보안상 출처(Origin)에 와일드카드()를 절대 사용할 수 없으며, 2번 항목의 명시적 출처만 적용됨.

---

### 📌 1-3 메서드 및 헤더 제어

- Allowed Methods:
    - `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `OPTIONS`
- Allowed Headers:
    - `Content-Type`
    - `Authorization`
    - `Idempotency-Key`
- Exposed Headers:
    - 현재는 Access Token을 응답 Body로 전달하므로 별도 노출 헤더 없음
    - 추후 파일 다운로드, 리다이렉트 등 필요 시 `Location`, `Content-Disposition` 추가 검토

---

### 📌 1-4 성능 최적화 정책 (Preflight Cache)

- **Max-Age 설정값:** `3600` (1시간)
- **목적:** 브라우저가 보내는 사전 요청(OPTIONS)의 결과를 1시간 동안 브라우저에 캐시하여, 중복된 예비 요청으로 인한 서버 API 부하를 최소화함.

## 2. Rate Limiting 정책

| 대상 API | 제한 기준 | 제한 횟수 | 초과 시 에러 |
| --- | --- | --- | --- |
| 로그인 | IP 기준 | 5회 / 1분 | COMMON_006 |
| 회원가입 | IP 기준 | 3회 / 10분 | COMMON_006 |
| 검색 | IP 또는 userId 기준 | 60회 / 1분 | COMMON_006 |
| 채팅 메시지 | userId 기준 | 30회 / 10초 | COMMON_006 |
| 파일 업로드 | userId 기준 | 10회 / 1분 | COMMON_006 |
| 결제 요청 | userId 기준 | 5회 / 1분 | COMMON_006 |

## 3. 비밀번호 정책

- 최소 8자 이상
- 영문, 숫자, 특수문자 포함
- 이메일/닉네임과 동일한 비밀번호 금지
- 연속된 숫자 사용 금지
- BCrypt로 단방향 암호화
- 비밀번호는 응답 JSON, 로그에 절대 출력하지 않음

## 4. 파일 업로드 제한 정책

| 항목 | 제한 |
| --- | --- |
| 허용 확장자 | jpg, jpeg, png, webp |
| 프로필 이미지 | 최대 5MB |
| 게시글 이미지 | 최대 5장, 장당 10MB |
| 리뷰 이미지 | 최대 5장, 장당 10MB |
| 영수증 이미지 | 필수, 최대 5MB |
| 채팅 파일 | 최대 10MB |
| 저장소 | AWS S3 |
| 파일명 | UUID 기반으로 변경 저장 |

## 5. CORS 설정 예시

```
java
@Configuration
public class CorsConfig {

    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration config = new CorsConfiguration();

        config.setAllowedOrigins(List.of(
                "http://localhost:3000",
                "http://localhost:5173"
        ));

        config.setAllowedMethods(List.of(
                "GET", "POST", "PUT", "PATCH", "DELETE", "OPTIONS"
        ));

        config.setAllowedHeaders(List.of(
                "Authorization",
                "Content-Type",
                "Idempotency-Key"
        ));

        config.setAllowCredentials(true);
        config.setMaxAge(3600L);

        UrlBasedCorsConfigurationSource source =
                new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/**", config);

        return source;
    }
}
```
