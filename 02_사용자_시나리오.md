## 0. 최종 사용자 흐름 보강

기존 시나리오는 초기 기획 기준이므로, 최종 구현 범위에 맞춰 다음 흐름을 추가 기준으로 둔다.

### 일반 사용자 핵심 흐름

1. 소셜 로그인 또는 일반 로그인으로 서비스에 진입한다.
2. 메인 검색창에서 장소·가게·메뉴를 통합 검색한다 (카테고리 구분: 장소 / 가게 / 메뉴).
3. 지도에서 주변 장소를 확인하고, 관광지·전통시장·찜한 장소 카테고리 및 지역 필터로 목록을 탐색한다.
4. 장소 상세 페이지에서 관광지 정보·주변 맛집·카페·숙박을 확인하고, 길찾기·찜하기·리뷰 작성·동행 모집글 확인 등의 기능을 이용한다.
5. 축제 탭에서 지역·월별 필터로 올해 남은 전국 축제를 탐색하거나, 캘린더로 오늘 진행 중인 행사를 확인한다.
6. 커뮤니티에서 동행 게시글 작성·참여 신청 후 채팅방으로 소통하거나, 자유 게시판에 사진·글을 올리고 댓글·대댓글로 소통한다.
7. 필요한 경우 채팅/고객센터 상담에서 번역 기능을 사용한다.
8. QR 결제 또는 스토어 주문으로 엽전을 사용하고, 마이페이지에서 내역을 확인한다.

### 상인 핵심 흐름

1. USER 계정으로 상인 신청을 등록한다.
2. 관리자가 승인하면 MERCHANT 권한으로 전환된다.
3. 내 가게 정보, 메뉴, 공지, 이미지, QR을 관리한다.
4. 고객의 QR 결제 요청을 승인/거절한다.
5. 매출과 가게 지갑을 확인하고 정산을 신청한다.
6. 광고를 신청하거나 연장한다.

### 관리자 핵심 흐름

1. 대시보드에서 사용자, 신고, 상인 신청, 정산, 광고 상태를 확인한다.
2. 신고를 검토하고 콘텐츠 숨김, 경고, 정지, 기각 등의 조치를 수행한다.
3. 축제/전통시장/관광지 데이터를 수집하거나 관리한다.
4. 환불, 정산, 광고, 상인 인증을 승인/거절한다.
5. 관리자 작업은 감사 로그로 남아 추적 가능하다.

## 1. 사용자 역할 정의

| 역할 | 설명 | 주요 관심사 |
| --- | --- | --- |
| `USER` | 일반 로그인 사용자 (내·외국인 여행자) | 관광지 탐색, 동행 매칭, 커뮤니티 |
| `MERCHANT` | 전통시장·관광지 주변 상인 | 가게 관리, 엽전 정산, 홍보 |
| `ADMIN` | 서비스 운영 관리자 | 사용자·상인 관리, 신고 처리, 콘텐츠 관리 |

## 2. 페르소나

### 페르소나 A — 외국인 여행자 “Emma”

```
이름      : Emma Johansson
나이      : 26세
국적      : 스웨덴
직업      : 프리랜서 그래픽 디자이너
여행 스타일: 계획 없이 현지 문화를 몸으로 느끼는 자유여행 선호
언어      : 영어 가능, 한국어 전혀 모름
기기      : iPhone, 데이터 로밍 사용
```

**배경**
한국 드라마를 보고 한국 문화에 빠진 Emma는 처음으로 혼자 한국 여행을 왔다.
서울의 유명 관광지는 이미 다녀왔고, 이제 현지인처럼 전통시장과 숨은 명소를 경험하고 싶다.
하지만 한국어를 전혀 몰라 길 찾기도, 시장에서 결제도 막막하다.
춘배투어 앱을 통해 동행 게시글을 발견하고, 현지인과 함께 광장시장을 방문하기로 한다.

**목표**
- 현재 위치 근처의 전통시장과 관광지를 바로 찾고 싶다
- 언어 장벽 없이 현지인과 소통하며 함께 다니고 싶다
- 현금 없이도 편하게 결제하고 싶다

**불편함 (Pain Points)**
- 구글 지도로는 전통시장 내부 가게 정보가 없다
- 카드가 안 되는 곳이 많아 현금을 항상 들고 다녀야 한다
- 현지인과 언어가 달라 채팅·소통이 어렵다
- 혼자 처음 가는 골목길은 무섭고 불안하다

### 페르소나 B — 국내 여행자 “지수”

```
이름      : 박지수
나이      : 29세
국적      : 대한민국
직업      : 직장인 (마케터)
여행 스타일: 주말 여행, SNS 공유 즐김, 새로운 경험 추구
언어      : 한국어
기기      : Android, 카카오맵 사용 중
```

**배경**
혼자 여행을 즐기지만 좋은 장소를 발견했을 때 함께할 사람이 없어 항상 아쉬웠던 지수.
외국인 친구를 사귀고 싶지만 일상에서 만날 기회가 없었다.
최근 전통시장에 관심이 생겼는데 어디서 뭘 먹어야 할지 정보가 너무 분산되어 있어 불편하다.
춘배투어에서 동행 게시글을 작성해 외국인 여행자와 함께 통인시장을 방문하기로 계획한다.

**목표**
- 전통시장과 관광지 정보를 한 앱에서 한 번에 확인하고 싶다
- 외국인 여행자와 함께 투어를 해보고 싶다
- 여행 후 동행 리뷰로 서로의 신뢰도를 남기고 싶다
- 여행 후기와 사진을 자유 게시판에 공유하고 싶다

**불편함 (Pain Points)**
- 관광지·시장 정보가 블로그, 지도, SNS에 분산되어 있다
- 외국인과 함께할 동행을 구하는 채널이 마땅치 않다
- 동행 약속을 잡아도 상대방의 신뢰성을 알 수 없다

### 페르소나 C — 전통시장 상인 “영호 씨”

```
이름      : 김영호
나이      : 52세
직업      : 광장시장 내 전통 포장마차 운영 (20년 경력)
기기      : Android, 카카오톡만 사용
디지털 역량: 낮음 (스마트폰 기본 사용 수준)
```

**배경**
외국인 손님이 부쩍 늘었지만 말이 안 통해 주문을 제대로 받지 못할 때가 많다.
카드 단말기를 쓰면 수수료가 너무 많이 나가 현금만 받았는데, 외국인은 현금이 없는 경우가 많다.
온라인에 가게를 홍보하고 싶지만 어떻게 해야 할지 모른다.

**목표**
- 외국인 손님에게도 쉽게 주문·결제를 받고 싶다
- 수수료 부담 없이 결제를 처리하고 싶다
- 내 가게를 앱에 등록해서 홍보하고 싶다

**불편함 (Pain Points)**
- 외국인과 언어 소통이 안 된다
- 카드 수수료가 부담스럽다
- 온라인 홍보 방법을 모른다

```
이름      : 춘배투어 운영팀
역할      : 서비스 전반 관리
주요 업무  : 상인 인증, 신고 처리, 콘텐츠 관리, 광고 관리
```

**목표**
- 불량 게시글·신고를 빠르게 처리하고 싶다
- 신뢰할 수 있는 상인에게 인증 마크를 부여하고 싶다
- 관광지·축제 정보를 최신 상태로 유지하고 싶다

## 3. 사용자 시나리오

### 시나리오 1 — Emma의 전통시장 탐방 (핵심 시나리오)

> **페르소나**: Emma (외국인 여행자)
**목표**: 현재 위치 근처 전통시장 찾기 → 엽전으로 결제 -> 가게 리뷰 작성
> 

```
[상황]
Emma는 지하철 종로3가역에서 내렸다.
사전 여행 전 구글지도로 알아보고 왔는데 정확한 시장 가게 위치 찾기가 힘들다.
춘배투어 앱을 열고 현재 위치 기반으로 주변을 탐색한다.
```

| 단계 | Emma의 행동 | 시스템 반응 |
| --- | --- | --- |
| 1 | 앱 실행 → 지도 화면에서 현재 위치 확인 | GPS 기반 주변 관광지·전통시장 마커 표시 |
| 2 | “광장시장” 마커 클릭 | 시장 상세 정보 (운영시간, 대표 메뉴, 사진, 리뷰) 표시 |
| 3 | 커뮤니티에서 “광장시장 같이 갈 사람?” 동행 게시글 발견 → 참여 신청 | 게시글 작성자에게 참여 신청 알림 발송 |
| 4 | 게시글 작성자가 참여 신청 수락 | Emma에게 수락 알림 발송 + 채팅방 초대 |
| 5 | 채팅방에서 집합 장소·시간 조율 | 실시간 메시지 + 번역 (한국어 ↔ 영어) |
| 6 | “길찾기” 버튼으로 시장까지 이동 | 카카오맵 앱으로 연결, 현재 위치 기준 경로 안내 |
| 7 | 시장 도착 후 “김영호” 가게 QR 스캔 | 가게·메뉴 정보 표시, 결제 화면 번역은 PG사 제공 기능 활용 |
| 8 | 엽전으로 결제 | 엽전 잔액 차감, 상인 결제 내역 반영 |
| 9 | 리뷰 작성 (별점) | 가게 상세 페이지에 리뷰 등록 |

---

### 시나리오 2 — 지수의 주말 여행 계획 

> **페르소나**: 박지수 (국내 여행자)
**목표**: 새로운 관광지 발견 → 동행 모집 → 동행 생성 → 동행 리뷰
> 

```
[상황]
지수는 이번 주말에 서울 근교 여행을 계획 중이다.
지난번에 북촌 한옥마을을 다녀왔으니 이번엔 다른 곳을 가고 싶다.
또한, 혼자 여행하기 보다 춘배투어를 통해 또래 외국 여성 일행을 구해서 여행할 예정이다.
```

| 단계 | 지수의 행동 | 시스템 반응 |
| --- | --- | --- |
| 1 | 앱 실행 → 검색창에 “서울 전통시장” 입력 | 인기 검색어 기반 자동완성 표시 |
| 2 | 검색 결과 탐색 | 검색 결과를 거리/평점/인기 기준으로 표시 |
| 3 | “통인시장” 상세 페이지 진입 | 리뷰, 주변 관광지 함께 표시 |
| 4 | 캘린더에서 이번 주말 축제 일정 확인 | 축제·이벤트 캘린더 조회 |
| 5 | 커뮤니티에 동행 게시글 작성 후 채팅방 개설 | 동행 게시글 등록, 채팅방 생성 및 참여 신청 오픈 |
| 6 | Emma가 게시글 확인 후 참여 신청 | 지수에게 참여 신청 알림 발송 |
| 7 | 지수가 참여 신청 수락 | Emma에게 참여 수락 알림 발송 + 채팅방 초대 |
| 8 | 채팅방에서 일정 조율 (번역 지원) | 실시간 메시지 + 번역 |
| 9 | 지수가 “동행 생성” 버튼으로 여행 기간 지정 | 동행 생성, 참여자 전원에게 동행 기간 공유 |
| 10 | 여행 종료 후 참여자 전원이 “동행 완료” 클릭 | 상대방에게 동행 리뷰 작성 요청 |
| 11 | 동행 리뷰 작성 (평점·코멘트) | 상대방 신뢰 지표 누적 (노쇼 여부·동행 태도 등 판단 가능) |

### 시나리오 3 — 영호 씨의 가게 관리 (상인 시나리오)

> **페르소나**: 김영호 (전통시장 상인)
**목표**: 가게 등록 → 상인 인증 신청 → 메뉴 관리 → 엽전 정산 확인
> 

```
[상황]
운영팀에게 앱 등록을 권유받은 영호 씨가 처음으로 앱에 가게를 등록한다.
```

| 단계 | 영호 씨의 행동 | 시스템 반응 |
| --- | --- | --- |
| 1 | 일반 회원가입 후 상인 등록 신청 | USER 계정 생성 후 사업자 정보와 가게 정보 입력 |
| 2 | 가게 정보 등록 (위치, 운영시간, 대표 메뉴, 사진) | 가게 정보 저장, 지도 마커 등록 |
| 3 | 상인 인증 마크 신청 | 운영팀 검토 대기 상태로 저장 |
| 4 | 운영팀이 서류 확인 후 승인 | 사용자 role이 MERCHANT로 변경되고 상인 페이지 접근 가능 |
| 5 | QR 코드 출력 (테이블 배치) | QR 결제용 코드 생성 |
| 6 | 엽전 결제 수신 후 정산 내역 조회 | 일별·월별 정산 현황 대시보드 |

---

### 시나리오 4 — 운영팀의 신고 처리 (관리자 시나리오)

> **페르소나**: 운영팀 (관리자)
**목표**: 신고된 게시글 검토 → 처리 → 상인 인증 관리
> 

| 단계 | 운영팀의 행동 | 시스템 반응 |
| --- | --- | --- |
| 1 | 관리자 대시보드 접속 | 신고 건수, 신규 상인 인증 요청 현황 표시 |
| 2 | 신고 내역 클릭 → 해당 게시글 확인 | 신고 사유, 신고자, 게시글 원문 표시 |
| 3 | 게시글 삭제 처리 | 게시글 비공개 처리 |
| 4 | 상인 인증 요청 검토 | 사업자 정보, 가게 사진 확인 |
| 5 | 인증 승인 | 상인 계정에 인증 마크 부여 |
| 6 | 관광지 정보 신규 등록 | 관광지 데이터 입력, 지도 마커 반영 |

### 시나리오 5 — 고객센터 실시간 상담 

> **페르소나**: 일반 사용자 / 운영 관리자
**목표**: FAQ로 해결되지 않는 문의를 실시간 상담으로 처리
> 

| 단계 | 사용자/관리자의 행동 | 시스템 반응 |
| --- | --- | --- |
| 1 | 사용자가 고객센터 진입 | FAQ 검색 화면 표시 |
| 2 | FAQ로 해결되지 않아 실시간 상담 요청 | 고객센터 상담방 생성 |
| 3 | 사용자가 메시지 전송 | `/pub/support/rooms/{supportRoomId}/messages`로 발행 |
| 4 | 관리자가 관리자 페이지에서 상담방 확인 | `/admin/support/rooms`에서 대기 상담 목록 조회 |
| 5 | 관리자와 사용자 실시간 상담 | `/sub/support/rooms/{supportRoomId}` 구독자에게 메시지 전달 |
| 6 | 관리자가 상담 종료 | 상담방 상태가 CLOSED로 변경 |

      마커·지역필터     인기·최근 검색어      캘린더 (날짜별 행사)  게시글 작성·댓글·신고
         │               │                    │               │
      [장소 상세]    [장소 상세]          [축제 상세]     [게시글 상세]

      동행 모집글      가게 QR 결제                          └ 장소/축제 링크

      (카카오맵 연결)

         ┌───────────────┬────────────────────┐

      [채팅]          [스토어]             [마이]

    │
