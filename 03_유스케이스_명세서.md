## 1. 액터 정의

| 액터 | 설명 |
| --- | --- |
| `USER` | 일반 로그인 사용자 (내·외국인 여행자) |
| `MERCHANT` | 관리자 승인으로 상인 권한을 부여받은 사용자 (가게 관리, 메뉴 관리, 엽전 정산) |
| `ADMIN` | 서비스 운영 관리자 |
| `System` | 내부 자동화 처리 (Redis, 알림, 번역 등) |


### UC-10 위치 기반 탐색

| 항목 | 내용 |
| --- | --- |
| **유스케이스 ID** | UC-10 |
| **유스케이스명** | 위치 기반 탐색 |
| **액터** | USER |
| **목적** | 현재 위치 주변의 관광지·전통시장을 지도에서 탐색한다 |
| **사전 조건** | 로그인 상태, 위치 권한 허용 |
| **사후 조건** | 지도에 주변 장소 마커가 표시된다 |

**기본 흐름**

| 단계 | 사용자 | 시스템 |
| --- | --- | --- |
| 1 | 앱 실행 후 지도 탭 클릭 | - |
| 2 | - | GPS로 현재 위치 좌표 획득 |
| 3 | - | Redis Geospatial로 반경 N km 내 장소 조회 |
| 4 | - | 관광지·전통시장·축제 마커 지도에 표시 |
| 5 | 마커 클릭 | - |
| 6 | - | 장소 요약 팝업 표시 (이름, 카테고리, 평점) |

**대안 흐름**

- `2a` 위치 권한 거부 시 → 전체 목록 조회 화면으로 전환, 위치 기반 기능 비활성화
- `3a` 반경 내 장소 없을 시 → “주변에 등록된 장소가 없습니다” 안내 메시지 표시

**예외 흐름**

- `2e` GPS 오류 시 → 마지막 저장 위치 사용, 없으면 서울 중심 좌표(기본값) 사용

**대안 흐름**

- `4a` 검색 결과 없을 시 → “검색 결과가 없습니다. 다른 키워드를 시도해보세요” 표시
